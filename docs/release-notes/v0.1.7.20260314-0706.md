# CodePilot1C EDT — Git, импорт из информационной базы и подключение любых провайдеров

Сборка: `0.1.7.20260314-0706`

Агент получил полноценную работу с Git прямо из чата, умеет создать новый EDT-проект из связанной информационной базы, и теперь без проблем подключается к провайдерам, которые не отдают список моделей (Qwen Coding Plan и подобные).

---

## Git из чата

Раньше для работы с Git нужно было переключаться в терминал или использовать встроенный Git-клиент Eclipse. Теперь агент делает это сам:

- **Просмотр** — статус, ветки, remotes, лог, diff summary. Всё read-only, ничего не сломает.
- **Операции** — init, clone, fetch, pull, push, создание веток, add, commit. Структурированные операции через API, не произвольные shell-команды.
- **Импорт после клонирования** — агент клонирует репозиторий и сразу импортирует найденный проект в workspace. Одна команда вместо цепочки ручных действий.

EDT-проект может лежать во вложенной папке внутри репозитория — агент корректно находит git root. На больших монорепозиториях `git status` больше не упирается в таймаут — переведён на облегчённый режим.

---

## Создание проекта из информационной базы

Новый инструмент `import_project_from_infobase` — агент берёт связанную с проектом информационную базу, выгружает конфигурацию через standalone server во временный XML-каталог и импортирует в новый EDT-проект. Весь поток, который раньше требовал ручного прохождения через визарды EDT, теперь выполняется из чата.

Зачем: типичная ситуация — нужно начать работу с базой, которая есть только как информационная база без проекта в Git. Раньше это был многошаговый ручной процесс. Теперь — один промпт.

---

## Подключение OpenAI-compatible провайдеров

Некоторые провайдеры (Qwen Coding Plan, локальные прокси, кастомные endpoint'ы) не поддерживают `GET /models` — возвращают 404, хотя `chat/completions` работает нормально. Раньше плагин показывал красную ошибку при настройке, хотя провайдер был рабочий.

Теперь: если `/models` недоступен — плагин предлагает ввести имя модели вручную, без паники. Подключение нестандартных провайдеров стало проще.

---

## Итого

Три сценария, которые раньше требовали выхода из IDE или ручных визардов, теперь доступны из чата: работа с Git, создание проекта из ИБ, подключение нестандартных провайдеров. EDT всё больше становится единой средой разработки, где агент закрывает рутину.
