# CodePilot1C EDT — агент теперь работает с расширениями конфигурации

Сборка: `0.1.7.20260221-0620`

До этого релиза агент мог создавать и изменять объекты только в основной конфигурации. Теперь он умеет работать с расширениями: создать проект расширения, заимствовать объекты из основной конфигурации, изменить формы и модули, проверить результат через диагностику EDT. Полный цикл доработки типовой без снятия с поддержки.

---

## Что умеет агент

### Управление расширениями

- Получить список расширений в проекте
- Создать новый проект расширения конфигурации
- Посмотреть объекты расширения и их состояния
- Заимствовать (adopt) объекты из основной конфигурации в расширение
- Управлять состоянием свойств расширяемых объектов

### Полный рабочий цикл

Подтверждён e2e-сценарий, который покрывает типичную задачу "доработать типовую через расширение":

1. Получить расширения проекта
2. Создать расширение (или выбрать существующее)
3. Заимствовать нужный объект из основной конфигурации
4. Расширить форму, изменить модуль, переопределить процедуры
5. Получить диагностику EDT по результату

Всё это агент делает сам через API платформы. Не через файловую систему, не через текстовые патчи - через валидированные операции EDT с контролем ошибок.

---

## Почему это важно

Расширения - основной способ доработки типовых конфигураций 1С без снятия с поддержки. До сих пор агент мог помочь только с конфигурациями "с нуля" или с уже снятыми с поддержки. Теперь он покрывает самый массовый сценарий: "нужно добавить кнопку / поле / обработчик в типовую, не трогая основную конфигурацию".

---

## Практический эффект

- Доработка типовых через расширения - агент создаёт расширение, заимствует объекты, вносит изменения, проверяет диагностику.
- Всё через API EDT - валидация встроена, сломать структуру расширения невозможно.
- Работает в том же agent loop что и остальные инструменты: можно в одном промпте попросить "создай расширение, добавь кнопку печати в документ Реализация, напиши обработчик".
